# 学习助手小程序

这是一个面向学习资料整理的微信小程序项目，当前形态是：

- 前端：微信小程序原生页面，使用 TypeScript、WXML、WXSS。
- 后端：本地 Flask 服务，提供图片 OCR 接口。
- 数据：小程序本地缓存，保存笔记、错题和 OCR 识别记录。
- OCR：优先使用 PaddleOCR，失败时自动切换 RapidOCR 兜底。

详细架构、页面逻辑、数据结构和接口说明见 [PROJECT_GUIDE.md](PROJECT_GUIDE.md)。

## 已实现功能

### 首页工作台

- 显示笔记数量、错题数量、待复习错题数量和 OCR 识别记录数量。
- 提供三个核心入口：图片识别、整理笔记、复习错题。
- UI 已按用户任务重新设计，突出“先识别，再整理，再复习”的使用路径。

### 图片识别

- 支持选择照片或截图。
- 使用 `wx.uploadFile` 上传图片到本地 Flask 后端。
- 后端返回识别文本、行列表、提示信息和实际使用的识别引擎。
- 识别结果可以保存为笔记。
- 识别结果可以保存为错题。
- 每次识别成功会自动写入本地 OCR 识别记录，后续创建错题时可以导入。
- 识别失败时显示后端返回的错误提示，方便定位 Flask、域名、OCR 依赖等问题。

### 学习笔记

- 支持新建、编辑、查看详情、删除。
- 支持按标题和内容搜索。
- 支持按分类筛选。
- 支持从 OCR 结果创建笔记。
- 保存带图片的笔记前会调用 `wx.saveFile`，避免只保存临时图片路径。
- 列表和详情页已按阅读和管理场景重新设计。

### 错题本

- 支持新建、编辑、查看详情、删除。
- 支持按关键词、科目、掌握状态筛选。
- 支持标记为已掌握或未掌握。
- 支持标签录入和展示。
- 错题创建有三个明确入口：
  - 从记录创建：从已保存笔记或 OCR 识别记录导入题目、答案、解析。
  - 拍照创建：分别拍照识别题目、答案、解析。
  - 手动创建：直接填写全部内容。
- 三种入口进入编辑页后显示不同说明和不同字段操作，避免用户混淆。

### 后端 OCR 服务

- `GET /ping`：健康检查。
- `POST /ocr`：接收图片并返回 OCR 结果。
- 支持图片格式：`.jpg`、`.jpeg`、`.png`、`.bmp`、`.webp`。
- 后端监听 `0.0.0.0:5000`，局域网内设备可以通过电脑 IP 访问。
- PaddleOCR 初始化或识别失败时，会尝试 RapidOCR。

### 工程质量

- 全局中文化：页面标题、导航栏、按钮、占位符、弹窗、Toast、分类、科目、状态、后端提示。
- 修复小程序单行 `input` 中文显示不全的问题。
- `wx.uploadFile` 已处理 HTTP 状态码、空响应和 JSON 解析失败。
- 已新增 `.gitignore`，忽略 `node_modules/`、Python 虚拟环境、缓存和上传临时文件。
- 已通过前端类型检查：

```powershell
npm run typecheck
```

## 本地运行

### 1. 安装前端依赖

```powershell
npm install
```

### 2. 安装后端依赖

```powershell
cd server
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

### 3. 启动后端

```powershell
cd server
venv\Scripts\activate
python app.py
```

启动成功后应看到类似输出：

```text
Running on http://127.0.0.1:5000
Running on http://10.29.45.6:5000
```

浏览器访问：

```text
http://127.0.0.1:5000/ping
http://10.29.45.6:5000/ping
```

预期返回：

```json
{
  "success": true,
  "message": "服务运行中"
}
```

### 4. 配置小程序后端地址

前端请求地址在：

```text
miniprogram/utils/request.ts
```

当前示例：

```ts
export const API_BASE_URL = 'http://10.29.45.6:5000'
```

如果电脑 IP 变化，需要把这里改成新的局域网 IP。

### 5. 微信开发者工具设置

本地调试阶段：

- 导入当前项目。
- 小程序根目录使用 `miniprogram/`。
- 在“详情 -> 本地设置”中关闭“校验合法域名、web-view、TLS 版本以及 HTTPS 证书”。
- 如果出现 `uploadFile 合法域名校验出错`，先检查上述设置和 `API_BASE_URL`。

## 当前限制

- 数据只保存在小程序本地缓存中，卸载或清缓存会丢失。
- 后端是本地开发服务，没有账号体系、鉴权和云端存储。
- 本地调试使用 HTTP，正式上线必须改为 HTTPS。
- PaddleOCR 在部分 Windows、Python、Paddle 版本组合下可能失败，当前依赖 RapidOCR 兜底保证识别流程可用。
- OCR 结果只做文本提取，没有自动区分题目、答案、解析，需要用户手动校对和拆分。
